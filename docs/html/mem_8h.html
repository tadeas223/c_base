◆ MemoryBaseCommitFunc

typedef none_Result MemoryBaseCommitFunc(MemoryBase *self, void *ptr, u64 size)

function commiting memory

virtual method for MemoryBase

should be used only for memory that was reserved earlier

common behaviour:

  • commits memory starting from ptr up to ptr + size
  • maps the reserved memory block to physical ram
  • makes the memory block useable
  • on error returns a result with result.ok = false
  • the returned error is defined in specific implementation of this function
Parameters
selfpointer to MemoryBase
ptrpointer to the start of the memory block
sizesize of the memory block to commit
Returns
result with a possible error

◆ MemoryBaseReserveFunc

typedef ptr_Result MemoryBaseReserveFunc(MemoryBase *self, u64 size)

function reserving memory

virtual method for MemoryBase

common behaviour:

  • reserves space in virual memory
  • does not map to physical ram
  • the space is unuseable until MemoryBaseCommit is called
  • returns a generated address of the reserved memory
  • if reservation fails error is returned
  • the error code and group is defined in the specific implementation of this function
Parameters
selfpointer to MemoryBase
sizesize of the desired memory space
Returns
result with pointer to the reserved memory or error

coppies memory from one place to another

coppies the size bytes from src to dest

if src > dest memory is coppied from left to right if dest > src memory is coppied from right to left

be careful with this function because when used incorrectly it can cause segmentation faults
